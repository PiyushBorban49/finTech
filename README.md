**Prerequisites**

- Node.js 18+ (LTS recommended)
- Git
- One of: `npm`, `yarn`, `pnpm`, or `bun`

**1) Clone the repo**

```bash
git clone <your-repo-url>
cd <your-repo-name>
```

**2) Install dependencies**

Use whichever package manager you prefer:

```bash
npm install
# or
yarn install
# or
pnpm install
# or
bun install
```

**3) Configure environment variables**

Create a `.env` file in the project root and add the keys below. Replace the example values with your real secrets.

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=
CLERK_SECRET_KEY=
GROQ_API_KEY=
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
```

How to obtain these keys:

- **Clerk**: Sign in at https://clerk.com → create an application → Dashboard → API Keys (Publishable & Secret).
- **Groq**: Sign in at https://app.groq.ai or the Groq Cloud Console → API Keys → Create API Key.
- **Supabase**: Sign in at https://supabase.com → create/select project → Project Settings → API → Project URL & anon key.

Keep `.env` out of version control (it's already in `.gitignore` by default).

**4) Supabase Setup**

To enable the eye-tracking logs and admin dashboard, you need to create a table in your Supabase project.

1.  Go to the **SQL Editor** in your Supabase Dashboard.
2.  Run the following SQL to create the `eye_tracking_sessions` table:

```sql
create table eye_tracking_sessions (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  duration integer,
  converted boolean,
  confusion_events jsonb,
  gaze_points jsonb,
  help_shown boolean
);
```

3.  (Optional) **Populate with Dummy Data**:
    If you want to test the Admin Dashboard without running through the session flow, run this SQL:

```sql
insert into eye_tracking_sessions (duration, converted, confusion_events, gaze_points, help_shown)
values
(12, true, '[{"zoneId": "price-summary", "reason": "Prolonged focus", "timestamp": 5000}]'::jsonb, '[{"x": 50, "y": 50}, {"x": 51, "y": 51}]'::jsonb, true),
(25, false, '[]'::jsonb, '[{"x": 10, "y": 10}, {"x": 20, "y": 20}]'::jsonb, false),
(45, true, '[{"zoneId": "terms-cond", "reason": "Rapid scanning", "timestamp": 12000}]'::jsonb, '[{"x": 80, "y": 80}, {"x": 85, "y": 85}]'::jsonb, true);
```

**5) Run the dev server**

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open http://localhost:3000 in your browser.

You can edit the app starting at [app/page.js](app/page.js). Changes reload automatically.

**Build / Production**

```bash
npm run build
npm start
```

**Troubleshooting**

- If you see auth or API errors, confirm your keys are set in `.env.local` and that you restarted the dev server after changing the file.
- If installs fail, delete `node_modules` and lockfile (`package-lock.json` / `yarn.lock` / `pnpm-lock.yaml`) then reinstall.
- Port conflicts: set `PORT` env var before starting, e.g. `PORT=3001 npm run dev` (Windows PowerShell: `$env:PORT=3001; npm run dev`).

**Deploying**

On Vercel, set the same environment variables in the project settings (do not upload `.env.local`).
